# Backend

FastAPI-бэкенд для системы управления потоковым вещанием из ТЗ хакатона.

## Что уже покрыто

- авторизация и регистрация с хешированием пароля;
- мульти-роли `Пользователь` / `Ведущий` / `Администратор`;
- soft delete пользователей;
- фильтрация и администрирование пользователей;
- сообщения ведущему и архив по статусам;
- голосовые сообщения пользователей;
- личная медиатека ведущего;
- плейлисты с порядком элементов, loop/shuffle, удалением и reorder;
- состояние эфира с запуском/остановкой, текущим треком и общей громкостью;
- WebSocket-контур для live-стрима;
- smoke-тест основного HTTP-сценария.

## Быстрый старт

1. Перейдите в каталог `backend`.
2. Создайте и заполните `.env` на основе `.env.example`.
3. Установите зависимости:

```bash
python -m venv venv
./venv/bin/pip install -r requirements.txt
```

4. Запустите сервер:

```bash
./venv/bin/python -m uvicorn main:app --reload
```

Сервер сам создаёт каталоги `storage/*`, роли по умолчанию и дефолтного администратора.

## Дефолтный админ

Задаётся через переменные окружения:

- `DEFAULT_ADMIN_LOGIN`
- `DEFAULT_ADMIN_PASSWORD`
- `DEFAULT_ADMIN_FIO`

По умолчанию это `admin / admin123`.

## Полезные команды

Запуск smoke-теста:

```bash
./venv/bin/python -m unittest backend.tests.test_api_smoke -v
```

Проверка синтаксиса:

```bash
./venv/bin/python -m compileall backend
```

## Контракт API

Основные группы ручек:

- `/api/auth` — логин, регистрация, текущий пользователь;
- `/api/admin` — пользователи, роли, смена пароля, soft delete;
- `/api/player` — статус эфира, сообщения, голосовые сообщения;
- `/api/host` — медиатека, плейлисты, эфир, входящие сообщения;
- `/api/stream` — WebSocket-стрим и статус live-подключений.

Дополнительно для ведущего доступны dev-friendly ручки:

- `POST /api/host/broadcast/next`
- `PUT /api/host/broadcast/volume`
- `PUT /api/host/broadcast/current-media`
- `DELETE /api/host/playlists/{playlist_id}/items/{item_id}`
- `PUT /api/host/playlists/{playlist_id}/items/reorder`
- `GET /api/host/voice-messages`

## Архитектурные заметки

- Пути к SQLite и `storage` нормализуются относительно каталога `backend`, поэтому запуск из разных cwd больше не ломает проект.
- Загрузка файлов идёт чанками, без чтения больших аудио/видео целиком в память.
- Для SQLite включены `foreign_keys` и `WAL`, чтобы локальная командная разработка была стабильнее.
- Текущее состояние эфира хранится отдельно от плейлистов, поэтому запуск/остановка эфира не сводится к одному флагу `is_active`.
