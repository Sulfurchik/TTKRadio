Кейс
Система управления потоковым вещанием.
Задание:

Необходимо создать систему управления потоковым вещанием с возможностями
авторизации, администрирования пользователей, воспроизведения аудиопотока для
слушателей и управления эфиром для ведущего.
Система должна состоять из 4 модулей:
1. Авторизация
2. Администрирование
3. Главная страница (Плеер)
4. Раздел ведущего

Модуль «Авторизация»

Необходимо реализовать окно авторизации с вводом логина и пароля. Авторизация
должна происходить по нажатию на кнопку «Вход», в случае если пользователь с таким
логином и паролем есть в системе. Данные пользователей должны храниться в локальной
базе данных.
Дополнительно необходимо предусмотреть механизм регистрации пользователя.
Для регистрации необходимо указать логин, ФИО и дважды ввести одинаковый пароль.
Логин должен содержать только латинские буквы, ФИО только русские, пароль может
содержать только латинские буквы, цифры, символы.
При регистрации пользователю устанавливается роль «Пользователь» по умолчанию.
Сценарии использования:
Вход в систему
· Пользователь вводит логин и пароль.
· Система проверяет корректность введенных данных сверяясь с локальной БД.
· При успешной авторизации система предоставляет доступ к функционалу согласно
роли пользователя.
Регистрация нового пользователя
· Пользователь заполняет форму регистрации.
· Вводит логин (только латинские буквы).
· Вводит ФИО (только русские буквы).
· Дважды вводит пароль (латинские буквы, цифры, символы).
· Система проверяет корректность заполнения полей.
· Создает учетную запись с ролью «Пользователь».
· Система обязательно должна хешировать пароль перед сохранением в базу данных.

Модуль «Администрирование»

В верхней части интерфейса необходимо расположить элементы перехода между
модулями, элемент для перехода в нужный модуль должен иметь пиктограмму и название
модуля.
Модуль должен иметь подраздел «Управление пользователями».
Раздел «Управление пользователями» должен содержать таблицу списка

пользователей и панель фильтрации.
Таблица списка пользователей должна содержать данные и заголовки. Заголовки
должны иметь:
· Логин пользователя
· ФИО пользователя
· Роль
· Дату регистрации
· Действия
Действия должны содержать кнопки:
· Редактировать
· Удалить
· Сменить пароль
· Назначить роли
При выборе действия «Редактировать» необходимо выводить модальное окно с
данными пользователя и предоставлять возможность редактировать их.
Логин должен содержать только латинские буквы, ФИО только русские, пароль может
содержать только латинские буквы, цифры, символы.
При выборе действия «Удалить» пользователь должен мягко удаляться, чтобы в случае
если он фигурирует в сообщениях или истории не привести к невозможности просмотра
данных.
При выборе действия «Сменить пароль» необходимо выводить модальное окно с
вводом пароля и подтверждением ввода.
В системе предусмотрены 3 роли:
· Пользователь
· Ведущий
· Администратор
Реализовать возможность мультивыбора ролей (одному пользователю может быть
назначено несколько ролей одновременно).
Только пользователь с ролью «Администратор» может назначать другим пользователям
роли «Администратор» или «Ведущий».
Панель фильтрации должна иметь фильтры по полям:
· Логин пользователя
· ФИО пользователя
· Роль
· Дата регистрации
Модуль должен быть доступен только роли «Администратор».
Сценарии использования:
Просмотр списка пользователей
· Администратор переходит в раздел «Управление пользователями».
· Система отображает таблицу со списком всех зарегистрированных пользователей.
· Таблица содержит следующие данные:
o Логин пользователя (латинские буквы).
o ФИО пользователя (русские буквы).
o Роль (Пользователь/Ведущий/Администратор, возможен множественный
выбор).
o Дата регистрации.
o Действия (кнопки «Редактировать», «Удалить», «Сменить пароль», «Назначить
роли»).

Назначение ролей пользователю
· Администратор выбирает пользователя и нажимает «Назначить роли».
· Система открывает модальное окно со списком доступных ролей.
· Администратор выбирает одну или несколько ролей (мультивыбор).
· Система сохраняет выбранные роли.
Удаление пользователя
· Администратор выбирает пользователя из таблицы и нажимает кнопку «Удалить».
· Система выполняет «мягкое удаление».
· Удаленный пользователь исчезает из таблицы, но данные в истории сохраняются.
Смена пароля пользователя
· Администратор выбирает пользователя и нажимает «Сменить пароль».
· Система открывает модальное окно для ввода нового пароля.
· Система сохраняет новый пароль.

Модуль «Главная страница (Плеер)»

В верхней части интерфейса необходимо расположить элементы перехода между
модулями (для авторизованных пользователей с соответствующими правами).
Модуль представляет собой интерфейс слушателя.
Центральным элементом должен быть плеер потокового аудио.
Плеер должен иметь элементы управления:
· «Воспроизвести»
· «Поставить на паузу»
· «Отрегулировать громкость» (ползунок)
Должна быть реализована форма обратной связи для написания сообщения ведущему.
Модуль должен быть доступен ролям «Пользователь», «Ведущий», «Администратор».
Сценарии использования:
Прослушивание аудиопотока
· Пользователь переходит на главную страницу.
· Видит интерфейс плеера.
· Нажимает «Воспроизвести».
· Система начинает воспроизведение аудиопотока.
· Пользователь может регулировать громкость.
Отправка сообщения ведущему
· Пользователь вводит текст в поле сообщения.
· Нажимает кнопку «Отправить».
· Система фиксирует сообщение и отправляет его в раздел ведущего.
Дополнительный функционал:
1. Воспроизведение видео, если ведущий включил видео в плейлист или запустил
видеотрансляцию (плеер автоматически переключается в режим видео или
показывает видеопоток).
2. Запись голосового сообщения пользователем и направление его ведущему (кнопка
записи, остановка, отправка).
Сценарии использования доп функционала:
Отправка голосового сообщения
· Пользователь нажимает кнопку «Записать голос».

Модуль «Раздел ведущего»

В верхней части интерфейса необходимо расположить элементы перехода между
модулями.
Модуль предназначен для управления эфиром.
Функционал управления аудио:
· Включение/выключение воспроизведения аудио (общий переключатель вещания).
· Управление аудио потоком через плейлист записей.
· Загрузка записей с ПК в личную медиатеку пользователя. Файлы сохраняются на
аккаунте ведущего постоянно (до ручного удаления).
· Возможность создания плейлиста из файлов личной медиатеки.
· Возможность зациклить плейлист (режим повторения).
· Возможность перемешивания позиций в плейлисте в случайном порядке (режим
Shuffle)
· Возможность записать аудио сообщение (с микрофона ведущего) и добавить его в
плейлист.
· Возможность включить трансляцию аудио во время основной трансляции
(микширование или переключение).
· Возможность управления громкостью трансляции (общая громкость эфира).
· Ограничения на загрузку: Максимальный размер аудио файла - не более 50 Мб.
Поддерживаемые форматы: MP3, WAV, OGG.
Управление сообщениями:
· Отображение сообщений от пользователей.
· Смена статусов сообщений:
o «Новый»;
o «В работе»;
o «Завершено»;
· Модуль должен быть доступен ролям:
o «Ведущий»;
o «Администратор».
Сценарии использования:
Управление плейлистом
· Ведущий переходит в раздел управления эфиром.
· Загружает аудиофайлы с ПК в медиатеку.
· Выбирает файлы из медиатеки и формирует конкретный плейлист (очередь
воспроизведения).
· Включает режим зацикливания при необходимости.
· Запускает воспроизведение для слушателей.
Запись и добавление в эфир
· Ведущий нажимает «Записать аудио».
· Система записывает звук с микрофона.
· Ведущий добавляет запись в текущий плейлист или сразу в эфир.
