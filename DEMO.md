## 🔧 БЫСТРЫЙ СТАРТ

### 1. Запуск Backend

```bash
cd /home/Flany/Hack/transcom-stream/backend
source venv/bin/activate
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**Backend доступен:**
- API: http://localhost:8000
- Swagger Docs: http://localhost:8000/docs
- Redoc: http://localhost:8000/redoc

### 2. Запуск Frontend (отдельный терминал)

```bash
cd /home/Flany/Hack/transcom-stream/frontend
npm install  # Только первый раз
npm run dev
```

**Frontend доступен:** http://localhost:3000

## 📋 СЦЕНАРИЙ ДЕМОСТРАЦИИ (5-7 минут)

### Шаг 1: Регистрация нового пользователя (1 мин)
1. Откройте http://localhost:3000/register
2. Заполните форму:
   - Логин: `viewer` (только латинские буквы)
   - ФИО: `Зритель Зрительнов` (только русские)
   - Пароль: `viewer123`
   - Подтверждение: `viewer123`
3. Нажмите "Зарегистрироваться"

### Шаг 2: Вход как Администратор (1 мин)
1. Перейдите на http://localhost:3000/login
2. Логин: `admin`, Пароль: `admin123`
3. После входа видна панель навигации с ролью "Администратор"

### Шаг 3: Администрирование (2 мин)
1. Нажмите "Админка" в меню
2. Покажите таблицу пользователей с фильтрами
3. Назначьте роль новому пользователю:
   - Нажмите 📋 (назначить роли)
   - Выберите "Ведущий" + "Пользователь"
   - Сохраните
4. Покажите мягкое удаление (кнопка 🗑️)

### Шаг 4: Раздел Ведущего (2 мин)
1. Выйдите из аккаунта админа
2. Войдите как `testuser` / `test123` (если есть роль Ведущий)
   ИЛИ назначьте роль Ведущий пользователю и войдите под ним
3. Перейдите в "Эфир"
4. Загрузите тестовый аудиофайл (MP3)
5. Создайте плейлист
6. Добавьте файл в плейлист
7. Нажмите "Начать вещание"

### Шаг 5: Плеер и сообщения (1 мин)
1. Выйдите и войдите как обычный пользователь (`viewer` / `viewer123`)
2. На главной странице покажите плеер
3. Отправьте текстовое сообщение ведущему
4. Отправьте голосовое сообщение (кнопка "Записать голос")

### Шаг 6: Обработка сообщений (1 мин)
1. Вернитесь к аккаунту Ведущего
2. В разделе "Эфир" покажите входящие сообщения
3. Измените статус на "В работе" → "Завершено"
4. Покажите архив сообщений

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Ошибка: "ModuleNotFoundError"
```bash
cd backend
source venv/bin/activate
pip install -r requirements.txt
```

### Ошибка: "npm not found"
Установите npm:
- Arch: `sudo pacman -S npm`
- Ubuntu: `sudo apt install npm`

### Порт 8000 занят
```bash
pkill -f uvicorn
# или
uvicorn main:app --port 8001
```

### База данных не создается
```bash
rm backend/database.db
# Перезапустите backend
```

### Frontend не подключается к API
Проверьте proxy в `frontend/vite.config.js`:
```javascript
proxy: {
  '/api': { target: 'http://localhost:8000' }
}
```

## 📊 API ENDPOINTS (кратко)

### Auth
- `POST /api/auth/login` - Вход
- `POST /api/auth/register` - Регистрация  
- `GET /api/auth/me` - Текущий пользователь

### Admin (требуется роль Администратор)
- `GET /api/admin/users` - Список пользователей
- `PUT /api/admin/users/:id` - Редактировать
- `DELETE /api/admin/users/:id` - Удалить
- `POST /api/admin/users/:id/password` - Смена пароля
- `POST /api/admin/users/:id/roles` - Назначить роли
- `GET /api/admin/roles` - Список ролей

### Player
- `GET /api/player/stream` - URL потока
- `POST /api/player/messages` - Отправить сообщение
- `POST /api/player/voice` - Голосовое сообщение
- `GET /api/player/broadcast-status` - Статус вещания

### Host (требуется роль Ведущий/Администратор)
- `GET /api/host/media` - Медиатека
- `POST /api/host/media/upload` - Загрузка файла
- `GET /api/host/playlists` - Плейлисты
- `POST /api/host/playlists` - Создать плейлист
- `POST /api/host/broadcast/start` - Начать вещание
- `POST /api/host/broadcast/stop` - Остановить вещание
- `GET /api/host/messages` - Сообщения
- `PUT /api/host/messages/:id/status` - Статус сообщения
